# EXPERIENCE.md — badwers-notify

Visual identity reference: `DESIGN.md` (all token paths below resolve against its frontmatter).

---

## Foundation

**Form-factor:** web (DOM-rendered). Библиотека встраивается в любую веб-страницу через `<script>` или npm-import. Никакого Shadow DOM — стили наследуются или переопределяются через CSS-классы.

**UI system:** отсутствует. Библиотека самодостаточна, не зависит от Bootstrap, Tailwind или других UI-фреймворков. Все стили задаются через инлайн-CSS в конфиге и CSS-классы, генерируемые библиотекой.

---

→ Composition reference: `.working/key-screens.html` (5 визуальных сцен, иллюстрирующих все режимы работы)

## Information Architecture

```
Stack Container (для каждой активной позиции)
└── Notification Card × N
    ├── Icon (слева)
    ├── Content
    │   ├── Title (опционально)
    │   └── Message
    └── Close Button (справа, опционально)
```

**Stack Container** — фиксированно позиционированный контейнер. По одному на каждую из 6 позиций. Создаётся лениво при первом вызове `show()` для данной позиции. `[ASSUMPTION: контейнер создаётся в `<body>`, не требует корневого элемента]`

**Notification Card** — DOM-элемент с анимацией появления/закрытия. Уникальный ID (uuid или счётчик) для программного закрытия.

---

## Voice and Tone

Библиотека не диктует микротекст — это ответственность разработчика. В документации рекомендуется:
- **Info:** нейтральные формулировки без восклицательных знаков.
- **Success:** краткое подтверждение («Сохранено», «Готово»).
- **Attention:** вежливое предупреждение («Сессия истекает»).
- **Warning:** прямая формулировка проблемы («Ошибка соединения»).

## Component Patterns

### Notification Card

**Анатомия (слева направо):**
1. **Icon container** — 20×20px, flex-shrink: 0. SVG-иконка, цвет зависит от типа.
2. **Content area** — flex: 1, overflow: hidden. Заголовок (опционально) + сообщение.
3. **Close button** — 20×20px, flex-shrink: 0. Отображается только при `closable: true`.

**Поведение:**
- При `title` — заголовок семиболдом 15px, сообщение обычным 14px.
- Без `title` — сообщение занимает всю высоту, шрифт 14px.
- При `customClass` — CSS-класс добавляется к корневому элементу карточки.

### Stack

**Типы стеков:**
- **Top** (right-top, left-top, center-top): flex-direction: column. Новая нотификация вставляется в конец (снизу). Старые уходят вверх.
- **Bottom** (right-bottom, left-bottom, center-bottom): flex-direction: column-reverse. Новая нотификация вставляется в конец (сверху, благодаря reverse). Старые уходят вниз.

**Емкость:** `maxStack` (default 10). При превышении самая старая нотификация удаляется без анимации.

## State Patterns

| Состояние | Визуал | Поведение |
|-----------|--------|-----------|
| **Empty** | Stack Container существует, но без дочерних элементов | Нет визуального вывода |
| **Visible (default)** | Полная непрозрачность, штатная позиция | Активен таймер autoclose (если включён) |
| **Hover** | Без визуальных изменений | Таймер autoclose приостанавливается (pauseOnHover) |
| **Closing** | Opacity → 0, сдвиг в направлении позиции | Анимация `animationDuration` ms, затем remove из DOM |
| **Closed** | — | Элемент удалён, стек перераспределён |
| **Mobile** | Ширина 360px → `calc(100vw - 32px)` | Отступ от края 16px, анимация без горизонтального смещения |

## Interaction Primitives

### IP-1: Show notification
1. Создаётся DOM-элемент карточки с переданными параметрами.
2. Находится или создаётся Stack Container для целевой позиции.
3. Элемент вставляется в стек (append для top, prepend для bottom).
4. Запускается CSS-анимация появления (slide-in + fade-in).
5. Запускается таймер autoclose (если `autoclose: true`).
6. Возвращается `NotifyInstance` с методом `close()` и полем `id`.

### IP-2: Close notification (manual)
1. Пользователь нажимает ×.
2. Элементу добавляется CSS-класс `closing`.
3. По окончании `animationDuration` элемент удаляется из DOM.
4. Вызывается колбэк `onClose` (если передан).
5. Оставшиеся нотификации плавно смещаются (gap закрывается).

### IP-3: Close all
1. Все Notification Card получают CSS-класс `closing`.
2. По окончании анимации все контейнеры очищаются.

### IP-4: Auto-close timer
1. После появления запускается `setTimeout(duration)`.
2. При hover мыши на карточке — `clearTimeout`, запоминается оставшееся время. `[ASSUMPTION: используется `mouseenter`/`mouseleave`, а не `hover` CSS]`
3. При mouseleave — таймер возобновляется на оставшееся время.
4. По срабатыванию — выполняется IP-2.

### IP-5: Dark mode
1. При инициализации библиотека проверяет `window.matchMedia('(prefers-color-scheme: dark)')`.
2. При совпадении применяются dark-токены цветов из DESIGN.md.
3. Слушатель `change` переключает тему на лету.
4. При `prefers-reduced-motion: reduce` все анимации (появление, закрытие, смещение стека) заменяются на мгновенное переключение opacity без transform.
4. `[ASSUMPTION: no manual dark/light toggle — только media query]`

## Accessibility Floor

- **role="alert"** на каждой отдельной карточке нотификации при её добавлении в DOM.
- **aria-live="assertive"** на карточках типа `warning`; для остальных типов `role="alert"` обеспечивает `aria-live="assertive"` по умолчанию. `[ASSUMPTION: без отдельного aria-live атрибута — role="alert" достаточно]`
- **aria-label** на кнопке закрытия — по умолчанию `"Закрыть уведомление"`, переопределяется через опцию `closeAriaLabel` в конфиге библиотеки.
- **Закрытие по Escape** — фокус на кнопке закрытия, нажатие Escape закрывает нотификацию. При закрытии с клавиатуры фокус перемещается на кнопку закрытия следующей нотификации в стеке; если нотификация последняя — на Stack Container.
- **prefers-reduced-motion** — при `@media (prefers-reduced-motion: reduce)` анимация slide/fade отключается: нотификация появляется и исчезает мгновенно (opacity 0→1, без transform).
- **Фокус-индикатор** — кнопка закрытия имеет `:focus-visible { outline: 2px solid currentColor; outline-offset: 2px; }` для видимой фокусировки при навигации с клавиатуры.
- **Цветовой контраст:** success-light (#fff на #1e8a3a) — 4.52:1, attention-light (#fff на #b85c00) — 4.60:1, warning-light (#fff на #dc3545) — 4.53:1, info-light (#222 на #fff) — > 14:1. Все dark-тема пары проходят WCAG AA.

## Key Flows

### KF-1: Разработчик показывает success-уведомление *(maps to UJ-1)*

**Протагонист:** Фронтенд-разработчик Дима, пишет форму регистрации.

**Entry state:** Форма отправлена, сервер вернул 200 OK.

**Path:**
1. Дима вызывает `BadwersNotify.success({ message: 'Регистрация прошла успешно' })`.
2. В правом верхнем углу плавно выезжает зелёная карточка с галочкой.
3. Через 5 секунд нотификация плавно исчезает (fade-out + slide-out).

**Climax:** Нотификация появилась, привлекла внимание, затем исчезла — интерфейс не заблокирован.

**Resolution:** Дима продолжает работу.

**Edge case:** Браузерный ad-blocker блокирует DOM-контейнер. Библиотека логирует предупреждение в console.warn, нотификация не показывается.

### KF-2: Разработчик показывает warning без автозакрытия *(maps to UJ-1 variant)*

**Протагонист:** Дима, обрабатывает критическую ошибку.

**Entry state:** Сервер вернул 500 Internal Server Error.

**Path:**
1. Дима вызывает `BadwersNotify.warning({ message: 'Сервер недоступен', autoclose: false })`.
2. Появляется красная карточка с крестиком, не исчезает.
3. Дима нажимает × — нотификация закрывается с анимацией.

**Climax:** Нотификация остаётся видимой до ручного закрытия — ошибка не теряется среди других уведомлений.

**Resolution:** Ошибка зафиксирована, уведомление закрыто. Дима продолжает исправлять проблему.

### KF-3: Стек из 3 нотификаций в позиции right-top *(maps to stack behavior)*

**Протагонист:** Дима, тестирует стековое поведение.

**Entry state:** Чистый экран, позиция `right-top`.

**Path:**
1. Дима вызывает `show({ message: 'A' })` — нотификация A появляется сверху.
2. Дима вызывает `show({ message: 'B' })` — B появляется ниже A.
3. Дима вызывает `show({ message: 'C' })` — C появляется ниже B.
4. Дима закрывает B (нажатием ×).

**Climax:** A и C плавно сдвигаются, gap от B заполняется с анимацией. Визуальный порядок после закрытия: A → C.

**Resolution:** Стек корректен, все нотификации на своих местах.

**Edge case:** Если вызвать 11-ю нотификацию при `maxStack: 10`, самая старая (A) удаляется без анимации, новая (K) добавляется внизу.

### KF-4: Стек из 3 нотификаций в позиции right-bottom *(maps to stack behavior)*

**Протагонист:** Дима, тестирует bottom-стек.

**Entry state:** Чистый экран, позиция `right-bottom`.

**Path:**
1. Дима вызывает `show({ message: 'A' })` — A появляется снизу.
2. Дима вызывает `show({ message: 'B' })` — B появляется выше A.
3. Дима вызывает `show({ message: 'C' })` — C появляется выше B.

**Climax:** Визуальный порядок (снизу вверх): A → B → C. Новые нотификации всегда сверху старых.

**Resolution:** Bottom-стек работает как ожидалось — свежие уведомления на виду.

### KF-5: Подключение через CDN *(maps to UJ-2)*

**Протагонист:** Фронтенд-разработчик Лена, делает лендинг на чистом HTML + JS.

**Entry state:** Пустой `<div id="app">` на странице, без сборщика.

**Path:**
1. Лена добавляет `<script src="badwers-notify.min.js">` в `<head>`.
2. Вызывает `BadwersNotify.config({ position: 'center-bottom' })`.
3. Вызывает `BadwersNotify.show({ message: 'Лендинг готов к демо' })`.

**Climax:** Нотификация появляется по центру внизу экрана — библиотека работает без npm, Vite, Webpack.

**Resolution:** Лена закрывает нотификацию и продолжает верстать.

### KF-6: Кастомизация стилей *(maps to UJ-3)*

**Протагонист:** Дизайнер-разработчик Катя, интегрирует библиотеку в брендированное приложение.

**Entry state:** Библиотека импортирована через npm, дефолтные цвета не подходят под brand-book.

**Path:**
1. Катя вызывает `BadwersNotify.config({ typeStyles: { success: { background: '#6f42c1', color: '#fff' } }, icons: { success: '<svg>...</svg>' } })`.
2. Вызывает `BadwersNotify.success({ message: 'Брендированное уведомление' })`.

**Climax:** Нотификация отображается с фирменным фиолетовым фоном и кастомной иконкой.

**Resolution:** Интеграция завершена, всё соответствует brand-book.

## Responsive & Platform

### Mobile (< 480px width)
- Ширина нотификации: 360px → `calc(100vw - 32px)` (отступы 16px с каждой стороны). `[ASSUMPTION: media-запрос `max-width: 480px`]`
- Стек: отступ от края экрана — 8px вместо дефолтного.
- Анимация: slide-in без смещения по горизонтали (выезжает сверху или снизу в зависимости от позиции).
